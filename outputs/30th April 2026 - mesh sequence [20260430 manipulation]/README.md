## Contents

- `scene.obj` — static helper: 1 m × 1 m table plane at Z=0, plus a 8 cm
  marker outline at the origin. Import once for visual context.
- `mesh_NNNN.obj` (542 frames at 30.0 fps, ~18.1 s) — per-frame
  hand meshes. Each file contains up to two named objects:
  `hand_slot0` (the leftmost-by-X hand in the source video) and `hand_slot1`.
  Frames where a hand wasn't tracked simply omit that object.

Hand meshes follow MANO topology: 778 vertices, 1538 triangles per hand.

## Coordinate frame

**Table frame, metric metres.** (0, 0, 0) is the marker centre on the
table. +Z is up from the table surface. Set your DCC tool's scene units
to metres.

## Open in Blender

### Easiest — Stop Motion OBJ addon

1. Install the [Stop Motion OBJ](https://github.com/neverhood311/Stop-motion-OBJ)
   addon (free, in the Blender extension menu or downloadable from GitHub).
2. File → Import → Mesh Sequence...
3. Point at this directory, set the file pattern to `mesh_*.obj`, and
   set the playback frame rate to **30 fps**.
4. Import `scene.obj` separately (File → Import → Wavefront (.obj)) for
   the table + marker reference.

### Manual via Python (no addon)

Paste into Blender's Python console:

```python
import bpy
from pathlib import Path

seq_dir = Path("/absolute/path/to/this/directory")
for f in sorted(seq_dir.glob("mesh_*.obj")):
    bpy.ops.wm.obj_import(filepath=str(f))

# Then keyframe visibility per frame manually, or animate via the
# Mesh Cache modifier on a base mesh.
```

## Animation timing

Frame `mesh_NNNN.obj` corresponds to time `NNNN / 30` seconds since the

Set the scene render fps to **30** to play back at real-time speed.

## What's where

The hand meshes are oriented anatomically: looking down at the table
from above, with +Z pointing up at the camera. The marker on the table
is at (0, 0, 0). Hand positions on the table appear in the +X / +Y
plane (approximately; the marker's chirality defines the +X axis of the
table).

This is the **K-anchored metric mesh** from the canonical pipeline —
positions are stereo-correct (the wrist is metric), but the mesh size is
uniformly scaled by the per-frame `k ≈ 0.78` factor so the wrist matches
stereo. To get the WiLoR-monocular-sized mesh instead (slightly larger),
rebuild from `mano_pose` + `mano_betas` in the parent `.npz` via the
MANO model, then translate by `wrist_table[i, slot]`.
